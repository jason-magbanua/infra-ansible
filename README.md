# infra-ansible

Ansible playbooks and roles for configuring servers provisioned by Terraform across Proxmox, GCP, and OCI environments.

---

# Overview

This repository contains Ansible automation used to configure infrastructure created by the **infra-terraform** project.

Terraform is responsible for provisioning infrastructure such as:

- Virtual machines
- Networks
- Disks
- Cloud resources

Ansible then connects to those machines to configure the operating system, install software, and apply consistent system configuration.

Together they form a simple and reliable Infrastructure-as-Code workflow:

```
Terraform → Provision Infrastructure
        ↓
Ansible → Configure Systems
        ↓
Applications → Deploy Services
```

---

# Supported Infrastructure Providers

The infrastructure configured by this repository may be provisioned from:

- **Proxmox VE**
- **Google Cloud Platform (GCP)**
- **Oracle Cloud Infrastructure (OCI)**

Terraform generates the infrastructure and Ansible configures the systems after provisioning.

# Requirements

- Ansible ≥ 2.15
- Python ≥ 3.10
- SSH access to target servers
- Terraform-generated inventory

Recommended installation using a Python virtual environment:

```
python3 -m venv ansible-venv
source ansible-venv/bin/activate
pip install ansible ansible-lint
```

---

# SSH Access

Servers are provisioned with the **infra user**.

Example SSH connection:

```
ssh infra@server-ip
```

The SSH key is injected during VM provisioning through Terraform and cloud-init.

# Typical Workflow

## 1. Provision Infrastructure

Infrastructure is created using the `infra-terraform` repository.

```
terraform apply
```

This creates servers across supported providers.

Terraform also generates the Ansible inventory.

---

## 2. Run Ansible Bootstrap

Bootstrap playbooks prepare the server with base configuration.

```
ansible-playbook -i inventories/generated/hosts playbooks/bootstrap.yml
```

Typical bootstrap tasks include:

- installing system packages
- enabling QEMU guest agent
- configuring SSH
- preparing base system settings

---

## 3. Apply System Configuration

Run the base configuration playbook:

```
ansible-playbook -i inventories/generated/hosts playbooks/base.yml
```

This may configure:

- firewall
- monitoring agents
- Docker or container runtimes
- system services

# Design Principles

This repository follows several core principles:

### Separation of Responsibilities

```
Terraform → infrastructure provisioning
Ansible → system configuration
```

Each tool focuses on its intended purpose.

---

### Reusable Roles

Configuration logic should live inside **roles** rather than large playbooks.

Roles should be:

- modular
- reusable
- environment agnostic

---

### Idempotent Automation

Playbooks should be safe to run repeatedly.

Running the same playbook multiple times should not break systems or introduce changes unless necessary.

---

### Minimal Complexity

Avoid unnecessary abstraction or over-engineering.

The goal is a simple and maintainable automation environment.
