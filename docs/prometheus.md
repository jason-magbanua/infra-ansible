# Prometheus

Ansible role that installs and configures Prometheus v3 as a native binary (no Docker) on a dedicated LXC container. Scrapes node_exporter from all infrastructure hosts, Traefik metrics, and cAdvisor from the Docker host. Config reloads are done via HTTP (`/-/reload`) without a service restart.

---

## Architecture

```
Prometheus LXC (10.10.200.20:9090)
  ├── node_exporter  → all LXCs + docker-host (:9100)
  ├── traefik        → Traefik metrics (10.10.200.10:8080/metrics)
  └── cadvisor       → Docker host containers (10.10.200.50:8080/metrics)
```

Data is stored at `/var/lib/prometheus` with a 15-day retention window.

---

## Playbook

```bash
cd ansible/
ansible-playbook playbooks/LXC/setup-prometheus.yml
```

The playbook targets the `prometheus` inventory group. No vault file is required — Prometheus has no secrets.

---

## Inventory

The `prometheus` host must be in `inventory/lab/hosts` under `[proxmox_containers]`:

```ini
[proxmox_containers]
prometheus   ansible_host=10.10.200.20
```

> **Note:** This file is auto-generated by `terraform apply`. Add the host to the Terraform ansible inventory template (`terraform/ansible.tf`) so it persists across applies.

## Role Variables

### `defaults/main.yml` — override these per environment

| Variable                  | Default          | Description                                    |
|---------------------------|------------------|------------------------------------------------|
| `prometheus_version`      | `3.2.1`          | Prometheus release (no `v` prefix)             |
| `prometheus_install_dir`  | `/usr/local/bin` | Directory for `prometheus` and `promtool`      |
| `prometheus_config_path`  | `/etc/prometheus`| Config directory                               |
| `prometheus_storage_path` | `/var/lib/prometheus` | TSDB data directory                       |
| `prometheus_retention`    | `15d`            | Storage retention window                       |
| `prometheus_port`         | `9090`           | Web UI / API listen port                       |
| `alertmanager_ip`         | `10.10.200.21`   | Alertmanager address                           |
| `alertmanager_port`       | `9093`           | Alertmanager port                              |
| `traefik_ip`              | `10.10.200.10`   | Traefik metrics host                           |
| `traefik_metrics_port`    | `8080`           | Traefik API/metrics port                       |
| `cadvisor_ip`             | `10.10.200.50`   | cAdvisor host                                  |
| `cadvisor_port`           | `8080`           | cAdvisor port                                  |
| `node_exporter_port`      | `9100`           | Port scraped on all node_exporter targets      |
| `node_exporter_targets`   | (list)           | List of `{host, ip}` dicts — see below         |

### `vars/main.yml` — internal, not meant to be overridden

| Variable                | Value                                        |
|-------------------------|----------------------------------------------|
| `prometheus_user`       | `prometheus`                                 |
| `prometheus_binary_url` | GitHub releases URL derived from version     |
| `prometheus_archive_dir`| `prometheus-{{ version }}.linux-amd64`       |

### Adding or removing scrape targets

Edit `node_exporter_targets` in `defaults/main.yml` and re-run the playbook. The template loops over this list; no other file needs to change.

## File Layout on the Target Host

```
/usr/local/bin/prometheus       # main binary
/usr/local/bin/promtool         # config validation tool
/etc/prometheus/
└── prometheus.yml              # main config (restart not needed on change)
/var/lib/prometheus/            # TSDB data (owned by prometheus user)
/etc/systemd/system/prometheus.service
```

## Handlers

| Handler                  | Triggered by                        | Action                                 |
|--------------------------|-------------------------------------|----------------------------------------|
| `reload prometheus config` | `prometheus.yml` change           | `POST http://localhost:9090/-/reload`  |
| `restart prometheus`     | Binary update or unit file change   | `systemctl restart prometheus`         |
| `reload systemd`         | Unit file change                    | `systemctl daemon-reload`              |

Config changes use the HTTP reload endpoint (`--web.enable-lifecycle`) — no service interruption. Binary and unit file changes trigger a full restart.

---

## Systemd Hardening

| Directive          | Value                      | Effect                                        |
|--------------------|----------------------------|-----------------------------------------------|
| `NoNewPrivileges`  | `true`                     | Prevents privilege escalation                 |
| `PrivateTmp`       | `true`                     | Isolated `/tmp`                               |
| `ProtectSystem`    | `strict`                   | Read-only filesystem except explicit paths    |
| `ReadWritePaths`   | `/var/lib/prometheus`      | Allows TSDB writes only                       |

Prometheus does not need privileged ports (port 9090), so no capability grants are needed.

## Traefik Metrics

Traefik exposes a Prometheus metrics endpoint at `:8080/metrics` when the following is present in `traefik.yml` (already added to `roles/traefik/templates/traefik.yml.j2`):

```yaml
metrics:
  prometheus: {}
```

This shares the API port (`8080`). The Traefik dashboard at `traefik.magbanua.xyz` is unaffected — that runs on port 443 via the file provider.

---

## cAdvisor

cAdvisor must be installed separately on `docker-host` (10.10.200.50) before the scrape job is useful. The Prometheus scrape config is already present — it will produce no data until cAdvisor is running. No special metrics_path configuration is needed; cAdvisor serves at `/metrics` by default on port 8080.

## Adding a New Scrape Job

1. Add a new `job_name` block to `roles/prometheus/templates/prometheus.yml.j2`
2. Re-run the playbook — `promtool check config` validates the config before any reload

Example:

```yaml
  - job_name: my_service
    static_configs:
      - targets:
          - "10.10.200.X:PORT"
        labels:
          instance: my_service
```

If the target list is variable per environment, add a variable to `defaults/main.yml` and loop over it in the template (see the `node_exporter` job for the pattern).

## Common Pitfalls

- **`promtool check config` runs before the reload handler fires** — a bad config will fail the play before Prometheus ever sees it. Fix the template, then re-run.
- **`--web.enable-lifecycle` is required for HTTP reload** — it is set in the service unit. Do not remove it or the `reload prometheus config` handler will return 403.
- **`ProtectSystem=strict` + TSDB writes** — `/var/lib/prometheus` is in `ReadWritePaths`. If you change the storage path in `defaults/main.yml`, update the unit template too, or Prometheus will fail to write data.
- **Alertmanager scrape section is a placeholder** — it points to `10.10.200.21:9093` even if Alertmanager isn't deployed yet. Prometheus will log a scrape error until Alertmanager is up; this is harmless.
- **cAdvisor job returns no data until cAdvisor is running** — the scrape config is present and valid; the target will just show as `down` in the Prometheus UI until the service is deployed.
