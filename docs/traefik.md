# Traefik Reverse Proxy

Ansible role that installs and configures Traefik v3 as a native binary (no Docker) on a dedicated LXC container. Handles TLS via Cloudflare DNS-01 challenge (wildcard cert), hot-reloads dynamic service config, and secures the dashboard behind basic auth.

---

## Architecture

```
Internet → Traefik LXC (10.10.200.10)
               ├── vault.magbanua.xyz          →  Vaultwarden   (10.10.200.41:80)
               ├── wiki.magbanua.xyz           →  Wiki.js        (10.10.200.43:3000)
               ├── grafana.magbanua.xyz        →  Grafana        (10.10.200.22:3000)
               ├── pve1.magbanua.xyz           →  Proxmox UI     (172.16.1.8:8006)
               ├── jellyfin.magbanua.xyz       →  Jellyfin       (docker-host)
               ├── plex.magbanua.xyz           →  Plex           (docker-host)
               ├── jackett.magbanua.xyz        →  Jackett        (docker-host)
               ├── sonarr.magbanua.xyz         →  Sonarr         (docker-host)
               ├── radarr.magbanua.xyz         →  Radarr         (docker-host)
               ├── qbittorrent.magbanua.xyz    →  qBittorrent    (docker-host)
               ├── flaresolverr.magbanua.xyz   →  FlareSolverr   (docker-host)
               └── traefik.magbanua.xyz        →  Dashboard      (api@internal)
```

- Entrypoint `web` (port 80) redirects all traffic to `websecure` (port 443)
- TLS certificates are issued via Let's Encrypt DNS-01 through Cloudflare and stored in `/etc/traefik/acme.json`
- Dynamic service configs live in `/etc/traefik/conf.d/` and are hot-reloaded — no service restart needed when adding or changing routes

## Playbook

```bash
cd ansible/
ansible-playbook playbooks/LXC/setup-traefik.yml --ask-vault-pass
```

The playbook targets the `traefik` inventory group and loads `secrets/traefik-vault.yml` automatically.

The `traefik` host must be present in `inventory/lab/hosts` under `[proxmox_containers]`:

## Secrets — `secrets/traefik-vault.yml`

Must be Ansible Vault encrypted before committing.

```yaml
traefik_cf_api_token: "your-cloudflare-dns-api-token"
traefik_dashboard_password: "admin:$2y$05$..."   # htpasswd bcrypt format
```

Generate the dashboard password:

```bash
# Install htpasswd if needed
sudo apt install apache2-utils

# Generate bcrypt hash (prompts for password)
htpasswd -nB admin
```

Encrypt the vault file:

```bash
ansible-vault encrypt secrets/traefik-vault.yml
```

> The Cloudflare token is written **only** to `/etc/traefik/traefik.env` (mode 0600) and injected into Traefik via `EnvironmentFile=`. It never appears in `traefik.yml`.

## Handlers

| Handler           | Triggered by                                    |
|-------------------|-------------------------------------------------|
| `reload systemd`  | systemd unit file change                        |
| `restart traefik` | `traefik.yml` change, unit file change, or new binary |

`conf.d/` changes do **not** trigger a restart — Traefik hot-reloads them via `watch: true`.

---

## Systemd Hardening

The service unit applies the following restrictions:

| Directive              | Value                    | Effect                               |
|------------------------|--------------------------|--------------------------------------|
| `NoNewPrivileges`      | `true`                   | Prevents privilege escalation        |
| `PrivateTmp`           | `true`                   | Isolated `/tmp`                      |
| `ProtectSystem`        | `strict`                 | Read-only filesystem except explicit paths |
| `ReadWritePaths`       | `/etc/traefik/acme.json` | Allows cert writes only              |
| `AmbientCapabilities`  | `CAP_NET_BIND_SERVICE`   | Allows binding ports 80/443 as non-root |
| `CapabilityBoundingSet`| `CAP_NET_BIND_SERVICE`   | Caps elevated capabilities to just the above |

## Adding a New Service

1. Create a new template in `roles/traefik/templates/conf.d/<service>.yml.j2`
2. Add a deploy task in `roles/traefik/tasks/main.yml` (no handler needed — hot-reload handles it)
3. Re-run the playbook

Every router must include the `domains:` block so Traefik serves the wildcard cert rather than requesting an individual cert per hostname. Traefik deduplicates — the wildcard is only fetched/renewed once regardless of how many routers reference it.

```yaml
http:
  routers:
    myservice:
      rule: "Host(`myservice.{{ traefik_domain }}`)"
      entryPoints:
        - websecure
      service: myservice
      tls:
        certResolver: cloudflare
        domains:
          - main: "{{ traefik_domain }}"
            sans:
              - "*.{{ traefik_domain }}"

  services:
    myservice:
      loadBalancer:
        servers:
          - url: "http://10.10.200.X:PORT"
```

## Common Pitfalls

- **`acme.json` permissions** — must be `0600` before the service starts or Let's Encrypt will refuse to write certificates. The role creates the file with correct permissions; do not change them manually.
- **`traefik_version` must include the `v` prefix** — the GitHub release URL format requires it (e.g. `v3.3.4`, not `3.3.4`).
- **Cloudflare token scope** — the token needs `Zone → DNS → Edit` permission scoped to your zone. A token with broader permissions works but violates least privilege.
- **Every router must include the `domains:` block** — without it, Traefik falls back to requesting an individual cert for that hostname instead of reusing the wildcard. All three `conf.d/` files include this block; any new service you add must too.
- **`ProtectSystem=strict` + log files** — if you later enable file-based logging in `traefik.yml`, add the log path to `ReadWritePaths` in the service unit, or the service will fail to write logs.
