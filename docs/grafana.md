# Grafana

Ansible role that installs Grafana via the official APT repository (not Docker or snap) on a dedicated LXC container. Provisions Prometheus as the default datasource, pre-loads three dashboards, and exposes the UI through Traefik at `grafana.magbanua.xyz`.

---

## Architecture

```
Internet → Traefik (10.10.200.10)
               └── grafana.magbanua.xyz → Grafana LXC (10.10.200.22:3000)
                                              └── datasource: Prometheus (10.10.200.20:9090)
```

---

## Playbook

```bash
cd ansible/
ansible-playbook playbooks/LXC/setup-grafana.yml --ask-vault-pass
```

The playbook targets the `grafana` inventory group and loads `secrets/grafana-vault.yml` automatically.

Verify after the run:

```bash
curl -s http://10.10.200.22:3000/api/health
# {"database":"ok"}
```

## Inventory

The `grafana` host must be in `inventory/lab/hosts` under `[proxmox_containers]`:

```ini
[proxmox_containers]
grafana   ansible_host=10.10.200.22
```

> **Note:** This file is auto-generated by `terraform apply`. Add the host to the Terraform ansible inventory template (`terraform/ansible.tf`) so it persists across applies.

## Secrets — `secrets/grafana-vault.yml`

Must be Ansible Vault encrypted before committing.

```yaml
grafana_admin_password: "your-admin-password"
```

Encrypt:

```bash
ansible-vault encrypt secrets/grafana-vault.yml
```

The password is written only to `grafana.ini` (mode 0640, group grafana). It never appears in logs or provisioning files.

## Datasources

Provisioned datasources are **read-only in the Grafana UI** — this is expected behavior for file-provisioned datasources.

| Datasource    | UID           | Status      | URL                              |
|---------------|---------------|-------------|----------------------------------|
| Prometheus    | `prometheus`  | Active       | `http://10.10.200.20:9090`       |
| Loki          | `loki`        | Placeholder  | Commented out in datasources.yml |
| Alertmanager  | `alertmanager`| Placeholder  | Commented out in datasources.yml |

To activate Loki or Alertmanager: uncomment the relevant block in `roles/grafana/templates/provisioning/datasources.yml.j2` and re-run the playbook.

## Dashboards

Dashboards are downloaded from grafana.com to the control machine on first run (`roles/grafana/files/dashboards/`) and copied to the target host on every run. Datasource input variables (`${DS_PROMETHEUS}`) are rewritten to explicit UIDs (`prometheus`) so file-provisioned panels resolve without manual mapping.

| Dashboard           | Grafana ID | File                       |
|---------------------|------------|----------------------------|
| Node Exporter Full  | 1860       | `node-exporter-full.json`  |
| Traefik             | 17346      | `traefik.json`             |
| cAdvisor            | 14282      | `cadvisor.json`            |

All dashboards appear in the **Homelab** folder in the Grafana UI.

To add a dashboard:
1. Add an entry to `grafana_dashboards` in `roles/grafana/vars/main.yml`
2. Re-run the playbook

## Traefik Integration

A conf.d entry was added to the Traefik role at `roles/traefik/templates/conf.d/grafana.yml.j2`. It is deployed automatically when the Traefik playbook runs. No extra middlewares — Grafana handles its own authentication.

Re-run the Traefik playbook to pick up the route if Traefik was already deployed before Grafana:

```bash
ansible-playbook playbooks/LXC/setup-traefik.yml --ask-vault-pass
```

## Common Pitfalls

- **Dashboard datasource `${DS_*}` variables** — Grafana's file provisioner does not process `__inputs` substitution. The role rewrites these to explicit UIDs using `ansible.builtin.replace`. If a newly added dashboard uses a different input variable name, the regex `DS_[A-Z0-9_]+` will still match it correctly.
- **Cached dashboard JSONs** — `force: false` on the download task means grafana.com is only fetched once per file. To pull a newer dashboard revision, delete the file from `roles/grafana/files/dashboards/` and re-run.
- **Loki and Alertmanager panels will error** — dashboards that include Loki panels will show "datasource not found" until Loki is deployed and the datasource entry is uncommented. This is expected.
- **cAdvisor dashboard shows no data** — cAdvisor must be running on `docker-host` (10.10.200.50:8080) for the dashboard to populate. The datasource scrape config in Prometheus is already present.
- **APT key dearmoring** — the role downloads the Grafana GPG key as ASCII-armored and converts it with `gpg --batch --yes --dearmor`. If `gnupg` is not installed on the LXC, the role installs it first.
